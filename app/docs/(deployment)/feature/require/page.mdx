# Request Database

ฟีเจอร์ **Request Database** เป็นส่วนเสริมที่ช่วยให้ผู้ใช้งานสามารถขอเปิดใช้งานฐานข้อมูล **MySQL** เพิ่มเติมได้ในภายหลัง
โดยระบบถูกออกแบบมาให้มีการตรวจสอบสิทธิ์ผ่านผู้ดูแลระบบ (Admin) ก่อนการสร้างฐานข้อมูลจริง เพื่อการจัดสรรทรัพยากรที่มีประสิทธิภาพและความปลอดภัยของข้อมูล

- **Database Status**: แสดงสถานะปัจจุบัน (เช่น `Connected` หรือ `Not Connected`)
- **Request Database Account**: ปุ่มสำหรับส่งคำร้องขอสร้างฐานข้อมูลใหม่

## Workflow

กระบวนการทำงานของระบบขอใช้งานฐานข้อมูล แบ่งเป็นขั้นตอนดังนี้:

1. **User Request**: ผู้ใช้งานกดปุ่ม `Request Database Account` ผ่านหน้า Profile
2. **Admin Review**: คำร้องจะถูกส่งไปยัง Dashboard ของผู้ดูแลระบบ (Admin)
3. **Approval**: เมื่อ Admin กดอนุมัติ ระบบจะทำการสร้าง Database Name, User และ Password ให้โดยอัตโนมัติ
4. **Activation**: สถานะของผู้ใช้จะเปลี่ยนเป็น `Connected` และสามารถนำ Account ของระบบเชื่อมต่อไปยัง PHP Myadmin ได้

<Callout type="info">
  **ข้อดีของระบบนี้** ช่วยลดความซับซ้อนในการจัดการ Database รายบุคคล และทำให้ Admin สามารถควบคุมจำนวน Database ที่ถูกสร้างขึ้นใน Server ได้ง่ายขึ้น
</Callout>

<Callout type="default">
  เมื่อ **Reset Password** สามารถเข้าใช้ Mysql Account โดยใช้ Username และ **password** (รหัสผ่านคือข้อความ "password") สามารถ กด Change Password เพื่อเปลี่ยนรหัสผ่านเพื่อให้ของ Platform และ MySQL Account ตรงกันได้ 
</Callout>
