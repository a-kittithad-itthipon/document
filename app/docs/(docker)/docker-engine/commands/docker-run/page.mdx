# Docker Run

## What is Docker Run
คำสั่ง `docker run` เป็นคำสั่งหัวใจหลักที่ใช้สำหรับ "สร้างและเริ่มการทำงาน" ของ Container ใหม่จาก Image ที่กำหนด หาก Image นั้นยังไม่มีในเครื่อง Docker จะทำการ pull มาให้โดยอัตโนมัติ คำสั่งนี้มักใช้คู่กับ Option ต่างๆ เพื่อกำหนดพฤติกรรมของ Container เช่น การตั้งชื่อ, การ Map Port หรือการรันแบบ Background

## Basic Commands

### Run (Foreground)
คำสั่งพื้นฐานสำหรับการรัน Container โดยจะแสดงผลลัพธ์ (Logs) ออกมาทางหน้าจอทันที และเมื่อโปรแกรมทำงานเสร็จ Container จะหยุดตัวเอง (เหมาะสำหรับการทดสอบ)
```bash
docker run hello-world
```
ผลลัพธ์ที่ได้จะมีลักษณะคล้ายกับตัวอย่างต่อไปนี้:
```bash
Hello from Docker!
This message shows that your installation appears to be working correctly.
...
```


### Run (Detached)
การรัน Container ในโหมดเบื้องหลัง (Background Mode) โดยใช้ Option `-d` เพื่อให้ Container ทำงานต่อไปเรื่อยๆ โดยไม่ยึดหน้าจอ Terminal (มักใช้คู่กับ `-p` เพื่อเปิด Port)
```bash
docker run -d -p 8080:80 nginx
```
ผลลัพธ์ที่ได้จะมีลักษณะคล้ายกับตัวอย่างต่อไปนี้:
```bash
c8b9a0d1e2f3a4b5c6d7e8f9a0b1c2d3e4f5a6b7c8d9e0f1a2b3c4d5e6f7a8b9
...
```

### Run (Interactive)
การรัน Container พร้อมกับขอสิทธิ์ในการเข้าถึง Terminal ข้างใน (`-it`) เพื่อให้เราสามารถพิมพ์คำสั่งโต้ตอบกับ OS ภายใน Container ได้
```bash
docker run -it ubuntu
```
ผลลัพธ์ที่ได้จะมีลักษณะคล้ายกับตัวอย่างต่อไปนี้:
```bash
root@d9b100f2f636:/#
```
