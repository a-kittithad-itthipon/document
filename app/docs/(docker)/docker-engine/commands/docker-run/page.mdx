# Docker Run

คอนเทนเนอร์ **hello-world** ที่คุณรันในขั้นตอนก่อนหน้านี้เป็นตัวอย่างของคอนเทนเนอร์ที่ทำงานและหยุดทำงานหลังจากส่งข้อความทดสอบ คอนเทนเนอร์มีประโยชน์มากกว่านั้นมาก และสามารถโต้ตอบได้ ที่จริงแล้วมันก็คล้ายกับเครื่องเสมือน เพียงแต่ใช้ทรัพยากรน้อยกว่าเท่านั้น

ยกตัวอย่างเช่น ลองรันคอนเทนเนอร์โดยใช้ภาพ Ubuntu เวอร์ชันล่าสุด การใช้ สวิตช์ -iและ-t ร่วมกัน จะทำให้คุณสามารถเข้าถึงเชลล์แบบโต้ตอบภายในคอนเทนเนอร์ได้:

```bash
docker run -it ubuntu
```

หน้าต่างคำสั่งของคุณควรเปลี่ยนไปเพื่อแสดงให้เห็นว่าขณะนี้คุณกำลังทำงานอยู่ภายในคอนเทนเนอร์ และควรมีลักษณะดังนี้:
```bash
root@d9b100f2f636:/#
```
โปรดสังเกตหมายเลขประจำตัวคอนเทนเนอร์ในหน้าต่างคำสั่ง ในตัวอย่างนี้คือ **d9b100f2f636** คุณจะต้องใช้หมายเลขประจำตัวคอนเทนเนอร์นี้ในภายหลังเพื่อระบุคอนเทนเนอร์เมื่อคุณต้องการลบออก

ตอนนี้คุณสามารถรันคำสั่งใดก็ได้ภายในคอนเทนเนอร์ ตัวอย่างเช่น ลองอัปเดตฐานข้อมูลแพ็กเกจภายในคอนเทนเนอร์ คุณไม่จำเป็นต้องใส่คำนำหน้าคำสั่งใดๆ **sudo** เพราะคุณกำลังทำงานภายในคอนเทนเนอร์ในฐานะ ผู้ใช้ root :
```bash
apt update
```

จากนั้นติดตั้งแอปพลิเคชันใดก็ได้ลงไป ในที่นี้เราจะติดตั้ง Node.js:
```bash
apt install nodejs
```

ขั้นตอนนี้จะติดตั้ง Node.js ลงในคอนเทนเนอร์จากที่เก็บซอฟต์แวร์อย่างเป็นทางการของ Ubuntu เมื่อการติดตั้งเสร็จสิ้น โปรดตรวจสอบว่า Node.js ได้รับการติดตั้งเรียบร้อยแล้ว:
```bash
node -v
```

คุณจะเห็นหมายเลขเวอร์ชันแสดงอยู่ในเทอร์มินัลของคุณ:
```bash
v12.22.9
```

การเปลี่ยนแปลงใดๆ ที่คุณทำภายในคอนเทนเนอร์จะมีผลเฉพาะกับคอนเทนเนอร์นั้นเท่านั้น

หากต้องการออกจากคอนเทนเนอร์ ให้พิมพ์ **exit** ที่พร้อมท์
